# [1.0.92](https://github.com/Cloudzero/provision-account/compare/1.0.91...1.0.92) (2026-04-09)

> Add unified Terraform module for AWS account onboarding

### Features
* **New `cloudzero-aws` Terraform module** — unified module replacing separate `cloudzero-payer` and `cloudzero-resource` modules. Supports both management and member account types via `create_cur` / `cur_bucket_name` toggle.
* **CUR creation support** — automatically creates S3 bucket, bucket policy, and CUR report definition for management accounts.
* **IAM policy parity** — adds missing `AWSBillingReadOnlyAccess` and `CloudWatchReadOnlyAccess` managed policies, updates stale policy Sids to match CloudFormation templates.
* **Feature toggles** — individual IAM policy blocks (activity monitoring, container insights, cloudwatch metrics, reserved capacity, optimization hub, cloudformation access) can be disabled.
* **Outputs** — role ARN, role name, account ID, CUR bucket name/ARN, CUR report name.
* **Preconditions** — mutual exclusion of `create_cur`/`cur_bucket_name`, us-east-1 enforcement for CUR report creation.
* **Examples** — member account, management account (new CUR), management account (existing CUR), multi-account.

### Deprecations
* `cloudzero-payer` and `cloudzero-resource` modules are deprecated with migration instructions pointing to `cloudzero-aws`.
