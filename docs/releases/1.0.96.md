# [1.0.96](https://github.com/Cloudzero/provision-account/compare/1.0.95...1.0.96) (2026-05-12)

> Grant EKS and ECS describe / list access for container cost attribution

## New Features

### Container Orchestration API Access
* **New `CZTier1ContainerOrchestration20260511` statement** — adds `eks:Describe*`/`eks:List*` and `ecs:Describe*`/`ecs:List*` on `*`. The `ViewOnlyAccess` AWS managed policy covers a baseline of these APIs but does not include newer additions (EKS nodegroups, Fargate profiles, addons; ECS Service Connect and capacity-provider APIs). The new statement closes those gaps and future-proofs against further AWS API additions in either service.
* **Why this matters for SCAD** — AWS Split Cost Allocation Data lands as additional columns in the CUR. Correlating those rows back to a real workload requires the cluster name, nodegroup, Fargate profile, ECS service, or task definition that produced the cost. With this statement CloudZero can resolve SCAD line items to the actual orchestration objects without relying on workload-side instrumentation.
* **Applied to both account types** — added to `master_payer.yaml` and `resource_owner.yaml` (plus their JSON policy mirrors). Most clusters live in resource-owner accounts, but the statement is on both for parity since payer accounts can run workloads too.

## Deployment Notes

* Read-only additions — no mutation actions, no breaking changes. Existing stacks gain the new statement on next update; the role assumes the additional describe/list capabilities automatically.
* The deprecated `terraform/cloudzero-payer` and `terraform/cloudzero-resource` modules are not updated here; users on those modules should migrate to `cloudzero-aws` (see v1.0.93 release notes).
