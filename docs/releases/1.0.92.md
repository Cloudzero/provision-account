# [1.0.92](https://github.com/Cloudzero/provision-account/compare/1.0.91...1.0.92) (2026-04-20)

> Reorganize inline IAM policy into neutral tiers, expand coverage of AWS cost services, add IAM self-inspection for transparency, and normalize the Terraform role path

## Upgrade Steps

* **[ACTION REQUIRED for Terraform users]** The IAM role path changes from `/` to `/cloudzero/` to align with the CloudFormation templates. Because IAM role path is immutable, `terraform apply` will destroy and recreate the `cloudzero-access` role. Coordinate a re-provisioning window; the new role ARN will be emitted by Terraform after apply. CloudFormation users are unaffected — the CFN templates already used `/cloudzero/`.

## New Features

### Tiered Inline Policy
* **Reorganize statements by capability focus** — Every statement now carries a `CZTier0*`, `CZTier1*`, or `CZTier2*` Sid prefix so it is clear at a glance what each grant supports.
* **Tier0** — cost and usage data ingestion (CUR bucket, billing, reserved instances, savings plans).
* **Tier1** — cost optimization signals (Compute Optimizer, Cost Optimization Hub, Trusted Advisor, Health, pricing, organizations and tagging metadata, Container Insights logs).
* **Tier2** — operational visibility (CloudTrail, CloudWatch metrics, service quotas, CloudFormation, IAM self-inspection).

### Expanded Cost Service Coverage
* **AWS Savings Plans** (`savingsplans:Describe*/Get*/List*`) — commitment details and upcoming expirations for renewal planning.
* **AWS Compute Optimizer** (`compute-optimizer:Describe*/Get*`) — rightsizing recommendations directly in inline.
* **Cost Optimization Hub** — broadened from two explicit actions to `cost-optimization-hub:Get*/List*` for full recommendation coverage.
* **BCM Data Exports** (`bcm-data-exports:Get*/List*`) — required for customers on the newer CUR 2.0 / FOCUS export format.
* **BCM Pricing Calculator** (`bcm-pricing-calculator:Get*/List*`) — what-if scenario estimates saved in your account.
* **AWS Billing Conductor** (`billingconductor:Get*/List*`) — pro-forma billing groups for enterprise chargeback.
* **MAP Credits** (`mapcredits:List*`) — Migration Acceleration Program credit visibility.
* **Purchase Orders** (`purchase-orders:Get*/List*/View*`) — enterprise PO / invoice alignment.
* **Sustainability** (`sustainability:Get*`) — account-level carbon footprint summary for ESG reporting alongside cost.
* **Trusted Advisor + AWS Health** — carved out into a dedicated Tier1 statement.
* **`cloudtrail:LookupEvents`** — added explicitly (not matched by `Get*`/`List*`/`Describe*` wildcards).

### IAM Self-Inspection (Transparency)
* **New CloudZero-scoped IAM read access** — CloudZero can now read its own role and attached policies so you can see exactly what access has been granted.
* `iam:GetRole`, `iam:GetRolePolicy`, `iam:ListAttachedRolePolicies`, `iam:ListRolePolicies`, `iam:ListRoleTags`, `iam:SimulatePrincipalPolicy` — scoped to `arn:aws:iam::*:role/cloudzero/*`.
* `iam:GetPolicy`, `iam:GetPolicyVersion` — scoped to exactly the four AWS managed policy ARNs CloudZero attaches (`AWSBillingReadOnlyAccess`, `CloudWatchReadOnlyAccess`, `ComputeOptimizerReadOnlyAccess`, `ViewOnlyAccess`).

### Documentation
* **New `policies/README.md`** — conventions, Sid prefix scheme, and long-term direction for consolidating master-payer and resource-owner policies.
* **New "Data Access Scope" section in `README.md`** — anchored on the AWS Cost and Usage Report. Documents every area where permissions reach beyond CUR content, with per-area rationale. Explicit statement that no permission reads customer resource contents (object bodies, database records, application logs, secrets, EC2 disks).
* **Permission Categories table** expanded to include the newly-covered cost services.

## Bug Fixes

* **Correct malformed log-group ARN** — The resource ARN on the Container Insights log query statement was `arn:aws:logs:*:*:log-group::log-stream:*` (syntactically malformed and using the wrong resource type — both `logs:DescribeLogGroups` and `logs:GetQueryResults` operate on `log-group`, not `log-stream`). Corrected to `arn:aws:logs:*:*:log-group:*`.

## Improvements

* **Budgets moved from Tier0 to Tier1** — Budgets are a cost-management concern, not a data-ingestion floor.
* **Alphabetized actions** within each statement for cleaner diffs and reviews.
* **Terraform role path** normalized from `/` to `/cloudzero/` to match CloudFormation.

## Breaking Changes

* **Terraform only:** The `cloudzero-access` IAM role is destroyed and recreated on the next `terraform apply` because its `path` attribute changes from `/` to `/cloudzero/`. The new role ARN appears in Terraform outputs after apply. CloudFormation-deployed roles are unaffected.

## Deployment Notes

### CloudFormation
* Templates can be deployed immediately with a stack update. Existing roles continue to work; the tiered inline policy replaces the previous policy in place.

### Terraform
* Run `terraform plan` first to review the role recreation.
* After `terraform apply`, the new role ARN will be different — update any downstream references.
